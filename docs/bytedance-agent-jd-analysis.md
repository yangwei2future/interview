### ⚠️ 薄弱项（需要补强）

#### 1. JVM 深度（高优先级）
岗位要求"对JVM有深入理解和实践"，这是后端基础的硬指标。

```
缺失内容：
  - JVM内存模型（堆、栈、方法区）
  - GC算法（CMS/G1/ZGC）
  - 类加载机制
  - OOM排查与性能调优
```

#### 2. Spring 生态深度（高优先级）
岗位要求"精通Java及Spring生态"，目前没有系统学习。

```
缺失内容：
  - Spring Bean生命周期
  - 循环依赖（三级缓存）
  - AOP原理（JDK/CGLIB动态代理）
  - @Transactional原理与失效场景
  - Spring Boot自动装配
```

#### 3. Python 开发能力（中优先级）
岗位明确要求Python，用于Skill等脚本开发。

```
现状：简历未体现Python经验
建议：梳理是否有Python使用经验（数据处理、脚本等）
      如果有，补充到简历；如果没有，面试时如实说"基础使用"
```

#### 4. 可观测性体系（中优先级）
岗位要求熟悉 Logging、Metrics、Tracing。

```
缺失内容：
  - 分布式链路追踪（SkyWalking/Jaeger）
  - Metrics监控体系（Prometheus/Grafana）
  - 结构化日志设计
建议：结合项目经验，梳理你们平台的监控方案
```

#### 5. Agent 评估与测试机制（中优先级）
岗位明确提到"构建Agent的评估、测试与迭代优化机制"，这是差异化要求。

```
需要了解：
  - Agent评估指标（准确率、召回率、幻觉率）
  - Prompt评估框架（RAGAS等）
  - AB测试在Agent场景的应用
```

#### 6. 多 Agent 协作模式（低优先级，了解即可）
```
需要了解：
  - 多Agent通信协议（MCP已有基础）
  - 任务分发与结果汇总
  - Agent间状态共享
```

#### 7. 工作流调度（低优先级）
岗位提到 Airflow/DolphinScheduler。

```
现状：有分布式任务调度经验（指标知识库Job服务）
建议：梳理清楚你们用的是什么调度框架，能说清楚原理
```

## 五、面试高频问题预测

根据JD，面试大概率会问：

**Agent方向：**
- 你们的 Agentic RAG 架构是怎么设计的？
- NL2SQL 如何保证准确性？遇到过哪些问题？
- MCP 协议的工作原理是什么？
- 如何评估 Agent 的效果？

**后端基础：**
- JVM GC 了解吗？G1 和 CMS 的区别？
- Spring 如何解决循环依赖？
- 高并发场景下如何设计系统？

**大数据：**
- Flink CDC 的原理是什么？
- StarRocks 为什么查询快？（列式存储）
- 数据治理你们是怎么做的？

**系统设计：**
- 如何设计一个高可用的 Agent 服务？
- 多 Agent 协作如何保证任务一致性？
